The problem

Three ventures. Three data worlds.

Customer data lives in separate Freshdesk instances and legacy databases per venture — there is no single, trusted profile of a customer across ADRES.

Current state
  • ✗  3 independent Freshdesk instances
  • ✗  Siloed legacy databases per venture
  • ✗  No unified customer / lead profile
  • ✗  Manual, inconsistent data handling
  • ✗  AI and marketing automation underused
Target state
  • ✓  One centralized Zoho CRM instance
  • ✓  Clean, deduplicated golden-record profiles
  • ✓  Documented schema across Accounts, Leads, Deals
  • ✓  Automation and SLA rules per business unit
  • ✓  A sprint-ready backlog for the CX team
Proposed approach

A four-stage delivery methodology

01 / DISCOVER

Stakeholder discovery

Structured interviews across Product, Sales, Tech and Marketing in all 3 ventures.

02 / DEFINE

Data strategy

Audit legacy/Freshdesk data, define dedup rules and a zero-loss migration path.

03 / DESIGN

Zoho architecture

Schema, fields, automation, omnichannel and integration blueprints.

04 / DELIVER

Document and handover

Consolidated CRM requirement doc plus a prioritized sprint-ready backlog.

Working model & risk

Built to run without surprises

How it runs
Discovery workshops

60–90 min structured sessions per team/venture; notes shared within 24h.

Documentation standard

BRD/TRD templates for every deliverable, diagrams in a shared workspace.

Governance cadence

Weekly steering check-in with the sponsor plus async phase-gate updates.

Change control

Anything outside scope is logged and quoted via a simple change order.

Risks & mitigations
Conflicting priorities

Facilitated prioritization workshop with the sponsor.

Poor legacy data quality

Extended profiling window flagged early, billed only if material.

Delayed stakeholder access

Timeline shifts day-for-day, communicated immediately.

Scope creep into build/config

Routed through the change order process, not absorbed silently.
